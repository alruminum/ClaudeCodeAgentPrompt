1인 개발자. 소규모 프로젝트부터 자체 중형 서비스까지 직접 기획·개발.

- 앱 / 웹서비스 / 앱+웹 하이브리드 형태 모두 가능
- B2B 납품 형태도 있음 (예: 메모리 배틀 — 어딘가에 납품하는 프로젝트)
- 현재 목표: 메모리 배틀 프로젝트 자체의 성공보다, 이후 모든 프로젝트에 재사용 가능한 에이전트 시스템 구축이 우선
- 원하는 시스템 특성: 영속성 (프로젝트 간 이식), 자율성 (손이 덜 감), 생산성 극대화
- 하네스 루프·에이전트 협업 정교화가 현재 메인 작업
