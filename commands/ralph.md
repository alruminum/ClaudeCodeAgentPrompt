# Ralph Wrapper Skill

유저의 자연어 요청을 구조화된 Ralph Loop 프롬프트로 변환한다.

## 입력 파싱

유저 메시지에서 다음을 추출한다:
- **반복 횟수**: "N번", "N회", "N iterations", "N번만" → `--max-iterations N` (없으면 기본 3)
- **작업 내용**: 나머지 전체 텍스트

## 작업 유형 판단 (완료 기준 생성 전 필수)

작업 내용을 읽고 아래 두 유형 중 하나로 분류한다.

### 유형 A — Bounded Fix (경계가 있는 수정)
특징: 무엇이 잘못됐는지 이미 알고 있고, 고치면 끝나는 작업.
예: "X 버그 수정", "Y 파일을 Z 정책에 맞게 현행화", "특정 패턴 제거"

→ 완료 기준: 수정이 완료됐는지 확인하는 **상태 확인형** (grep/diff)
→ 루프 특성: 1~2회로 끝나는 게 정상.

### 유형 B — Open-ended Improvement (열린 개선)
특징: "더 좋게", "빠짐없이", "최대한 개선" — 끝이 명확하지 않은 작업.
예: "전체 문서와 코드 대조해서 불일치 수정", "코드 품질 최대한 개선", "테스트 커버리지 높여"

→ 완료 기준: **"더 이상 발견되는 문제가 없을 때"** 형태로 잡는다.
→ 루프 특성: 매 회차마다 전체를 다시 스캔해서 새로운 문제를 찾아야 루프의 의미가 생김.

## 구조화 규칙

작업 유형(A/B)을 먼저 판단한 뒤, 아래 템플릿으로 프롬프트를 작성한다.
작업 내용이 모호하더라도 최대한 의도를 추론해서 채운다. 추측이 불가능한 항목만 `[미정]`으로 남긴다.

```
## 목표
[작업 내용에서 핵심 목표를 1~3문장으로 명확히 기술]
[작업 유형: Bounded Fix / Open-ended Improvement]

## 제약사항
[작업 내용에서 언급된 제약. 없으면 일반 제약 추론:
- 기존 코드/파일 구조 유지
- 외부 의존성 추가 금지 (명시 없을 시)
- 등]

## 완료 기준
[작업 유형에 따라 다르게 작성:

Bounded Fix → 수정 완료 상태를 확인하는 항목:
  각 항목은 반드시 "어떤 동작으로 확인하는가"가 포함된 형태여야 한다.
  "~해야 한다", "~이어야 한다" 같은 서술형은 기준이 아님.
  grep/read/diff/run 등 구체적 확인 행위로 표현할 것.
  좋은 예:
  - [ ] grep "문제 패턴" 대상파일 → 0건
  - [ ] git diff → 변경 존재
  나쁜 예:
  - [ ] 코드가 정책과 일치해야 한다

Open-ended Improvement → 전체 재스캔 후 잔여 문제 0개:
  - [ ] [대상 전체]를 처음부터 다시 스캔 — 발견된 문제 항목 목록을 명시
  - [ ] 위 목록이 비어 있음 (이전 회차에서 찾은 항목 포함 누적 0건)
  ※ 이 유형은 max-iterations를 넉넉하게 (5회 이상) 잡을 것]

## 완료 선언 규칙
위의 완료 기준이 **모두** 충족됐을 때만 정확히 아래 텍스트를 출력한다:
RALPH_DONE

조건 미충족 시 절대 출력하지 않는다. 루프를 탈출하기 위해 거짓으로 출력하는 것은 엄격히 금지된다.
```

## 실행 절차

### 1단계: 구조화된 프롬프트 생성 후 유저에게 제시

프롬프트를 생성한 뒤 아래 형식으로 유저에게 보여준다:

```
---
**Ralph Loop 실행 설정**

- 최대 반복: N회
- 완료 시 조기 탈출: 활성화 (RALPH_DONE 출력 시)

**구조화된 프롬프트:**

[생성된 프롬프트 전체 출력]

---
이대로 실행할까요? 수정이 필요하면 말씀해 주세요.
```

### 2단계: 유저 확인 대기

- 유저가 "응", "ㅇㅇ", "ok", "고", "실행", "그래" 등 긍정 응답 → 3단계 진행
- 유저가 수정 요청 → 해당 부분만 수정 후 다시 1단계 출력
- 유저가 취소 → 종료

### 3단계: 임시 파일 생성 후 ralph-loop 호출

프롬프트에 `[`, `]`, `(`, `)` 등 shell 특수문자가 포함될 수 있으므로,
**반드시** 임시 파일 방식으로 호출한다. 세션 충돌 방지를 위해 `mktemp`로 고유 파일명을 생성한다.

1. Bash 도구로 고유 파일명 생성 (macOS mktemp는 .md suffix 미지원):
```bash
echo "/tmp/ralph_task_$(date +%s)_$RANDOM.md"
```

2. Write 도구로 프롬프트를 위 경로에 저장 (예: `/tmp/ralph_task_1775794108_17464.md`)

3. Skill 도구로 ralph-loop 호출 (짧은 참조 프롬프트 사용):

```
Skill: ralph-loop:ralph-loop
Arguments: "Read <TMPFILE> and execute the task described there" --max-iterations N --completion-promise "RALPH_DONE"
```

4. 루프 완료(RALPH_DONE 또는 max-iterations 도달) 후 Bash로 임시 파일 삭제:

```bash
rm -f <TMPFILE>
```
