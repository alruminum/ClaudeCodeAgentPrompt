# UX Loop Skill

유저의 디자인/UX 변경 요청을 구조화된 컨텍스트로 만들고, 부족한 정보는 역질문으로 채운 뒤 하네스 design 루프를 시작한다.

## 정보 추출

유저 메시지에서 다음 3가지를 추출한다:

| 항목 | 설명 | 예시 |
|------|------|------|
| **화면/컴포넌트** (WHERE) | 어느 화면이나 컴포넌트인가 | "게임 메인 화면", "랭킹 리스트" |
| **문제/요청** (WHAT) | 무엇이 이상하거나 어떻게 바꾸고 싶은가 | "너무 심심해", "칸이 안 맞아", "더 임팩트 있게" |
| **참고** (REF) | 스크린샷, 기준 시안, 원하는 방향 | 이미지 첨부, "레트로 아케이드 느낌" |

## 역질문 규칙

**화면/컴포넌트(WHERE)가 없으면** 반드시 물어본다.
문제/요청(WHAT)이 없으면 물어본다.
참고(REF)는 없어도 진행한다.

질문은 **한 번에 최대 2개**. 이미 말한 건 다시 묻지 않는다.

## 디자인 컨텍스트 구조화

정보가 충분하면 아래 형식으로 구성한다 (추론 항목에는 `(추론)` 표시):

```
[화면] <화면/컴포넌트>
[요청] <무엇을 어떻게 바꾸고 싶은가>
[참고] <있으면>
```

## 실행 절차

### 1단계: 디자인 컨텍스트 + 모드 제시

```
---
**UX Loop 실행 설정**

[화면] ...
[요청] ...
[참고] ...

**모드 선택:**
- DEFAULT (기본): 시안 1개 → 유저 직접 확인 (APPROVE/REJECT)
- CHOICE: 시안 3개 → design-critic 심사 → 유저 PICK

어떤 모드로 실행할까요? (기본값: DEFAULT)
---
```

### 2단계: 유저 확인 대기

- 긍정 응답 / 모드 미언급 ("응", "ㅇㅇ", "ok", "고", "그래", "ㅇ") → DEFAULT로 3단계 진행
- "3개", "choice", "골라볼게", "비교" 등 → CHOICE로 3단계 진행
- 수정 요청 → 수정 후 1단계 재출력
- 취소 → 종료

### 3단계: 하네스 design 루프 실행

Bash 도구로 실행한다:

**DEFAULT 모드:**
```bash
bash ~/.claude/harness/executor.sh design \
  --context "[화면] <화면> / [요청] <요청>" \
  --prefix mb
```

**CHOICE 모드:**
```bash
bash ~/.claude/harness/executor.sh design \
  --context "[화면] <화면> / [요청] <요청>" \
  --choice \
  --prefix mb
```

GitHub 이슈 번호를 유저가 언급했으면 `--issue <N>` 추가.
